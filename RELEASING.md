# Releasing e2e-engineering

Distribution surfaces: **npm package** (`e2e-engineering`), **Claude marketplace** (standalone repo), and the portable **AGENTS.md / Cursor** variants (shipped inside the npm package). Canonical source is `.claude/skills/e2e-engineering/`; everything in `dist/` is generated by `npm run build`.

---

## One-time setup

### 1. Rename the GitHub repo (optional, cosmetic)
- GitHub UI: Settings → rename `AI-Engineering` → `e2e-engineering`, or `gh repo rename e2e-engineering`.
- Old URLs auto-redirect. Then update the local remote:
  ```bash
  git remote set-url origin https://github.com/marcosesares/e2e-engineering.git
  ```

### 2. npm account
- Log in once locally: `npm login` (account: marcosesares@gmail.com).
- For CI: create an npm **automation token** (Publish scope) → add as repo secret `NPM_TOKEN`.

### 3. Marketplace repo (standalone)
- Create an empty public GitHub repo, e.g. `marcosesares/e2e-engineering-marketplace`.
- For CI auto-sync, set:
  - secret `MARKETPLACE_TOKEN` — GitHub PAT with `repo` scope on that repo.
  - variable `MARKETPLACE_REPO` — `marcosesares/e2e-engineering-marketplace`.
- If unset, the marketplace CI job no-ops (publish to npm still runs).

## Per-release

Bump the version in `package.json` first (keep it in sync with the tag).

### Automated (recommended)
```bash
# edit package.json "version" to X.Y.Z, commit
git tag vX.Y.Z
git push --tags
```
`.github/workflows/release.yml` then:
1. builds + `npm publish --access public` (needs `NPM_TOKEN`),
2. force-pushes the generated tree to the marketplace repo (needs `MARKETPLACE_TOKEN` + `MARKETPLACE_REPO`).

### Manual (no CI)
```bash
npm run build
npm publish                                                   # prepublishOnly re-runs build
npm run publish:marketplace -- --remote <git-url> --push --force
```

## Verifying a release
```bash
npm view e2e-engineering version            # npm has the new version
npx e2e-engineering@latest init --dry-run   # installer pulls the published package
```
Marketplace (in Claude Code):
```
/plugin marketplace add marcosesares/e2e-engineering-marketplace
/plugin install e2e-engineering@e2e-engineering
```

## Notes
- **Never hand-edit `dist/`** — edit `.claude/skills/e2e-engineering/` (canonical) and run `npm run build`. The `AGENTS.md` and Cursor `.mdc` are the exceptions: they are hand-authored editorial flattenings maintained directly in `dist/`.
- The marketplace mirror is **force-pushed** every release — it is fully generated, holds no human commits, so overwriting is safe.
- Bump the `version` field in `package.json`, `dist/marketplace/.claude-plugin/marketplace.json`, and `plugin.json` together for a clean release (npm enforces unique versions).
