## Component Interactions & Data Flow

### Brainstorming → TDD Flow

```
┌─ Skill Execution Engine ──────────────────────────────┐
│                                                       │
│ Input: "Let's build a React component"               │
│                                                       │
│ 1. TRIGGER DETECTION                                 │
│    └─ Intent: "design", intent_confidence: 0.95      │
│                                                       │
│ 2. TRIGGER MATCHING                                  │
│    └─ Matched skill: "brainstorming" (id: bstorm)    │
│                                                       │
│ 3. SKILL LOADING                                     │
│    └─ Load .claude/skills/brainstorming/SKILL.md     │
│       Phases: [explore, clarify, propose, design,    │
│                self-review, user-review, invoke-plans│
│                                                       │
│ 4. CONTEXT INJECTION                                 │
│    Codebase: React + TypeScript, Jest tests          │
│    Session: main branch, all tests green             │
│    Task: implementing feature "user profile page"    │
│                                                       │
│ 5. GATE CHECK                                        │
│    - Brainstorming design gate: N/A (design phase)   │
│    - No gates block entry                            │
│                                                       │
│ 6. RED FLAG CHECK                                    │
│    - Phase 1 (explore): red flags for assumptions    │
│    - No violations detected                          │
│                                                       │
│ 7. SKILL RENDER                                      │
│    Render: "Step 1: Explore project context"         │
│    - Read recent commits                             │
│    - Assess scope (decompose if >1 subsystem)        │
│    - Check patterns in existing codebase             │
│                                                       │
│ 8. PHASE EXECUTION (Steps 1-8)                       │
│    Agent executes brainstorming flow:                │
│    - Explores project (reads files, commits)         │
│    - Asks clarifying questions                       │
│    - Proposes 2-3 approaches                         │
│    - Writes design doc                               │
│    - Self-reviews                                    │
│    - Awaits user review → HARD GATE                  │
│    - User approves design doc                        │
│                                                       │
│ 9. CHECKPOINT SAVE                                   │
│    Saves: skill=brainstorming, phase=8, decisions={..} 
│    Artifacts: docs/.../design-profile-page.md       │
│                                                       │
│ 10. WORKFLOW ADVANCEMENT                             │
│    Brainstorming metadata includes:                  │
│    "next_skill": "writing-plans"                     │
│    └─ Load writing-plans skill → restart from Step 1 │
│                                                       │
│ (Later, when writing code:)                          │
│ Input: "Let's implement the profile page"             │
│                                                       │
│ 1. TRIGGER DETECTION (again)                         │
│    Intent: "implement"                               │
│                                                       │
│ 2. TRIGGER MATCHING                                  │
│    Matched skill: "test-driven-development"          │
│                                                       │
│ 3. SKILL LOADING                                     │
│    Load .claude/skills/test-driven-development/...   │
│                                                       │
│ 4. CONTEXT INJECTION                                 │
│    (same codebase, session, task context)            │
│                                                       │
│ 5. GATE CHECK                                        │
│    ✓ TDD RED Phase gate enforced:                    │
│      "Must write FAILING test first"                 │
│    └─ Gate passed (no violation yet)                 │
│                                                       │
│ 6. RED FLAG CHECK                                    │
│    Phase 1 (RED) red flags:                          │
│    - "I'll test after" → NOT detected (agent avoids) │
│    - "Code before test" → NOT detected               │
│                                                       │
│ 7. SKILL RENDER                                      │
│    Render: "RED Phase: Write failing test"           │
│    - One behavior per test                           │
│    - Clear name (describes requirement)              │
│    - Real code (minimal mocking)                     │
│    - Expected: test fails with expected failure      │
│                                                       │
│ 8. PHASE EXECUTION (RED)                             │
│    Agent writes test:                                │
│    describe('UserProfile', () => {                  │
│      it('renders user name', () => {                │
│        const user = { name: 'Alice' };               │
│        render(<UserProfile user={user} />);          │
│        expect(screen.getByText('Alice')).toBeInDocument(); │
│      });                                             │
│    });                                               │
│                                                       │
│    Test runs → FAILS (component doesn't exist)       │
│    ✓ RED phase verified                              │
│                                                       │
│ 9. CHECKPOINT SAVE                                   │
│    Saves: skill=TDD, phase=RED, test_written=true    │
│                                                       │
│ 10. WORKFLOW ADVANCEMENT                             │
│    → GREEN phase (write minimal code)                │
│    (cycle continues until DONE)                      │
│                                                       │
└─────────────────────────────────────────────────────┘
```

## Component State Diagram

```
INITIAL STATE
     ↓
BOOTSTRAP_LOADED
     ├─ Trigger rules registered
     ├─ Tool abstractions ready
     └─ Awaiting user input
     ↓
[User Message Received]
     ↓
INTENT_DETECTED (intent extracted)
     ├─ IF explicit /skill → override
     └─ ELSE → match against triggers
     ↓
SKILL_MATCHED (skill_id found)
     ├─ confidence < threshold? → ask user which skill
     └─ confidence >= threshold? → proceed
     ↓
SKILL_LOADED (SKILL.md parsed)
     ├─ Metadata extracted
     ├─ Phases unpacked
     └─ Gates, red flags indexed
     ↓
CONTEXT_INJECTED (codebase, session, user context)
     ├─ All context available for skill rendering
     └─ Ready for gate checks
     ↓
GATE_CHECK (hard gates evaluated)
     ├─ ALL gates OK? → proceed to RED_FLAG_CHECK
     └─ GATE VIOLATED? → ask for consent
         ├─ User confirms? → log override, proceed
         └─ User declines? → offer alternatives (different skill, manual action)
     ↓
RED_FLAG_CHECK (anti-patterns detected?)
     ├─ NO violations? → proceed to SKILL_RENDER
     └─ VIOLATION detected? → show table, ask to confirm (soft gate)
         ├─ User acknowledges? → proceed
         └─ User self-corrects? → proceed
     ↓
SKILL_RENDER (present workflow to agent)
     ├─ Phase name, steps, examples rendered
     └─ Ready for execution
     ↓
PHASE_EXECUTE (agent follows steps)
     ├─ Agent reads phase
     ├─ Requests tools (git, bash, file ops)
     └─ Reports completion
     ↓
CHECKPOINT_SAVE (state persisted)
     ├─ Phase recorded
     ├─ Decisions recorded
     └─ Artifacts recorded
     ↓
PHASE_COMPLETE
     ├─ Next phase in same skill? → PHASE_ADVANCE
     ├─ Next skill defined? → SKILL_ADVANCE (return to INTENT_DETECTED)
     └─ Workflow done? → IDLE
     ↓
IDLE (await next intent)
     └─ Session continues until user ends
```

## Component Responsibilities Summary

| Component | Input | Process | Output |
|-----------|-------|---------|--------|
| **Trigger Detection** | User message | Extract semantic intent | intent, confidence |
| **Skill Loader** | skill_id | Read & parse SKILL.md | skill metadata, phases, gates |
| **Context Injector** | codebase path, session id | Gather files, commits, state | context object (injected into agent) |
| **Gate Enforcer** | skill, current_state | Evaluate gate preconditions | gate_status (OK, VIOLATED, ASK_CONSENT) |
| **Red Flag Detector** | agent behavior, skill red_flags | Match behavior against patterns | violation[], confidence[] |
| **Skill Renderer** | skill, context | Format for readability | skill_text (rendered, highlighted) |
| **Checkpoint Saver** | session, decisions, artifacts | Serialize to JSON | checkpoint file (git-ignored) |
