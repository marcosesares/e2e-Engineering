# Map the codebase before grilling, and merge grill-me into grill-with-docs

The pre-implementation sequence was `grill-me → [map-codebase?] → [research?] → [prototype?] → to-prd`, with two grilling skills: **grill-me** (no-doc Karpathy brainstorm, drove pre-impl) and **grill-with-docs** (language reconcile, ran once at Implementation entry). Two problems surfaced after real runs.

First, **grilling ran before mapping**. On brownfield work the human was grilled about a feature before anyone had read the code it touches — so questions were uninformed and re-asked things the codebase already answered, and the gating of map-codebase itself lived *inside* grill-me (circular: the brainstorm decided whether to go learn what it should have known before brainstorming). Second, **two grills with overlapping jobs**: grill-me settled direction with no doc awareness, then grill-with-docs re-opened terminology at impl entry — a second interview, on a seam (the headless impl loop) that cannot actually interview a human anyway (ADR 0016 makes the impl phase headless under flight).

We resequence pre-impl to **`[map-codebase? (brownfield)] → grill-with-docs → [research?] → [prototype?] → to-prd`** and collapse the two grills into one. **map-codebase runs FIRST**, gated by `taskType` (brownfield runs it, greenfield skips it) — never by grilling. **grill-with-docs becomes the single grilling skill** (absorbs grill-me): doc-aware by construction — it reads CONTEXT.md, ARCHITECTURE.md, and (brownfield) the just-produced codebase-map §4 *before* asking, brainstorms direction Karpathy-style, reconciles language inline as terms crystallise, and gates research/prototype. It degrades cleanly on greenfield (docs are empty at first; it seeds CONTEXT.md as language emerges). Because language is settled here, **Implementation starts at to-issues with no grill step** — the old impl-entry grill-with-docs is deleted. The skill file moves from `impl/grill-with-docs.md` to `pre-impl/grill-with-docs.md`; `pre-impl/grill-me.md` is removed.

This refines ADR 0003 (composition — one fewer skill, not more) and ADR 0009 (map-codebase still scoped and conditional, just earlier). The cost: greenfield now invokes a "with-docs" skill against near-empty docs (cosmetic — it still grills), and a single skill carries both brainstorm and language-reconcile responsibilities (accepted; they were always the same conversation, artificially split across two phases). Per-slice gap-finding remains the slice gap-check's job (ADR unchanged) — Implementation never re-grills.
