# Write per-Task artifacts at the Task root from pre-impl start — no base-then-copy

ADR 0017 introduced the [[Task queue]] and moved each Task's body under `.e2e-engineering/tasks/<id>/`. But the pre-implementation sub-skills (map-codebase, research, to-prd, to-issues) predate the queue and hard-coded base paths — they wrote `prd.json`, `progress.txt`, `codebase-map.md`, `research.md`, `test-cases/` directly under `.e2e-engineering/`. The Task `<id>` was only decided at gate-1 ("write the Task body under `tasks/<id>/`"), so gate-1 had to **copy** the artifacts from base into the Task dir. The copy was real but the originals were never removed — every run left dirty duplicate prd/progress/test-cases at `.e2e-engineering/` root, which then confused phase detection (Step 1 reads base `prd.json`) and polluted the workspace.

Root cause: the Task root was established too late (gate 1), after the artifacts had already been written somewhere else (base).

We fix it by establishing the **Task root** ONCE, at pre-impl start. SKILL.md Step 1 now derives `<id>` (kebab feature slug) and `mkdir .e2e-engineering/tasks/<id>/` *before* the first artifact is produced (map-codebase is the first writer on brownfield). Every sub-skill — map-codebase, grill-with-docs, research, to-prd, to-issues, context-checkpoint — writes DIRECTLY into the Task root. Gate-1 queueing is reduced to appending the `queue.json` entry; there is nothing to copy because the body is already in place. "Task root" is now a named concept (`.e2e-engineering/tasks/<id>/` multi-Task, `.e2e-engineering/` single-Task legacy) defined in SKILL.md and CONTEXT.md, and every schema/skill references it instead of a hard-coded base path.

This refines ADR 0017 (the `tasks/<id>/` layout is unchanged; only *when* the root is fixed and *where* writes land changes). The cost: `<id>` must be chosen earlier, before the spec is fully formed — accepted, because the feature slug is known from the request at intake, and a slug can be renamed by moving one dir if it must change. The benefit: no duplicate state, no copy step, deterministic phase detection, and a clean `.e2e-engineering/` root holding only `queue.json` + `tasks/`.
