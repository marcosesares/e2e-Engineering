# Coding + testing standards live in a versioned constitution, not CLAUDE.md or per-guideline skills

ship-it keeps one versioned `constitution.md` combining the karpathy coding guidelines (think-before-coding, simplicity-first, surgical-changes, goal-driven-execution) and the qa testing principles (real interfaces, diagnosable failures, no hardcoded sleeps, scope discipline). It is injected into every implementation subagent's context so all parallel slices share identical rails. Concept borrowed from spec_kit's "constitution".

Rejected alternatives: folding into CLAUDE.md (mixes standards with project setup, unversioned, harness-specific) and keeping each karpathy guideline as a separately-invocable sub-skill (no single rails doc — a subagent might load some and miss others). The constitution is deliberately distinct from CONTEXT.md (glossary) and ADRs (point-in-time decisions): it is the standing contract for how code and tests are written, and it is versioned so changes are visible.
