## Q-SC-01 — Should BR-15 (orphan cleanup) check cross-file usage before removing a symbol? 🔴

**Context:** `surgical-changes/design.md:Alternative Flows`; `edge-cases.md:EC-01`

**Spec affected:** [`_reversa_sdd/surgical-changes/design.md`], [`_reversa_sdd/surgical-changes/requirements.md`]

**Question:** BR-15 says "remove imports/variables/functions that YOUR changes made unused." This is currently scoped to the current file only. If the LLM removes a function that appears unused locally but is imported by another file, it introduces a runtime error in another module. Should BR-15 be scoped to "symbols unused in the current file AND confirmed unused across the codebase"? And if cross-file usage cannot be confirmed, should the LLM mention the potential orphan instead of deleting?

**Impact:** A scope clarification would add a sentence to SKILL.md and change the test criterion for RF-05. Without it, there is a real regression risk when orphan cleanup spans module boundaries.
