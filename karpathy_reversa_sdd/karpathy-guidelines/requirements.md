## Overview

A Claude Code skill that encodes four behavioral principles designed to counteract well-known LLM coding failure modes. When installed — either as a plugin or by appending content to `CLAUDE.md` — these principles become active constraints applied to every LLM interaction in the target project. 🟢

---

## Responsibilities

- Provide the LLM with four named behavioral principles as active constraints 🟢
- Package those principles for distribution via two install paths (plugin, CLAUDE.md) 🟢
- Expose a Claude Code plugin manifest and marketplace listing 🟢
- Supply bilingual documentation (English + Simplified Chinese) 🟢
- Serve as an opinionated default for projects that want production-quality LLM coding behavior 🟡

## Acceptance Criteria

```gherkin
# Happy path — plugin install
Given a user has Claude Code installed
When they run `/plugin marketplace add forrestchang/andrej-karpathy-skills`
And run `/plugin install andrej-karpathy-skills@karpathy-skills`
Then all four behavioral principles become active for the target project

# Happy path — CLAUDE.md install
Given a project has an existing CLAUDE.md
When the user appends the content of SKILL.md to it
Then all four behavioral principles become active for that project

# Failure — malformed plugin manifest
Given plugin.json has a schema error
When Claude Code attempts to install the plugin
Then the install fails with a validation error and no principles are activated

# Failure — translation divergence
Given SKILL.zh.md has been modified independently of SKILL.md
When a Chinese-language user installs the skill
Then the behavioral rules applied may differ from the English version
# Note: no automated guard exists for this — see questions.md 🔴
```
