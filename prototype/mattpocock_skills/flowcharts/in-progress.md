## review — two-axis parallel review

```mermaid
flowchart TD
    START([/review]) --> FIXED{Fixed point provided?}
    FIXED -- yes --> DIFF["Capture: git diff <fixed-point>...HEAD (three-dot)"]
    FIXED -- no --> ASK_POINT["Ask: review against what — branch, commit, or main?"]
    ASK_POINT --> DIFF

    DIFF --> SPEC_SRC[Identify spec source]
    
    subgraph SPEC_LOOKUP["Spec source (priority order)"]
        SP1[Issue refs in commits: #123 / Closes #45]
        SP2[Path argument from user]
        SP3["PRD/spec file under docs/, specs/, .scratch/"]
        SP4[Ask user]
        SP5[No spec → skip Spec axis]
        SP1 --> SP2 --> SP3 --> SP4 --> SP5
    end

    SPEC_SRC --> STD_SRC[Identify standards sources]

    subgraph STD_LOOKUP["Standards sources"]
        ST1["CLAUDE.md, AGENTS.md, CONTRIBUTING.md, CONTEXT.md"]
        ST2["docs/adr/ (decisions are standards)"]
        ST3["eslint.config.*, biome.json, .editorconfig (note, don't re-check)"]
        ST4["STYLE.md, STANDARDS.md, STYLEGUIDE.md"]
    end

    STD_SRC --> PARALLEL[Spawn PARALLEL sub-agents - single message, two Agent calls]

    subgraph STANDARDS_AGENT["Standards sub-agent"]
        SA1[Read standards docs]
        SA2[Read diff]
        SA3[Report violations per file/hunk]
        SA4[Cite file + rule for each]
        SA5[Distinguish hard violations from judgment calls]
        SA6[Skip tooling-enforced checks]
        SA7[Under 400 words]
        SA1 --> SA2 --> SA3 --> SA4 --> SA5 --> SA6 --> SA7
    end

    subgraph SPEC_AGENT["Spec sub-agent"]
        SP_A1[Read spec]
        SP_A2[Read diff]
        SP_A3["Report: a) missing requirements"]
        SP_A4["b) scope creep (unexpected behavior)"]
        SP_A5["c) wrong implementation of spec requirement"]
        SP_A6[Quote spec line for each finding]
        SP_A7[Under 400 words]
        SP_A1 --> SP_A2 --> SP_A3 --> SP_A4 --> SP_A5 --> SP_A6 --> SP_A7
    end

    PARALLEL --> STANDARDS_AGENT & SPEC_AGENT

    STANDARDS_AGENT & SPEC_AGENT --> AGG[Aggregate results]
    AGG --> REPORT["## Standards\n[verbatim findings]\n\n## Spec\n[verbatim findings]"]
    REPORT --> SUMMARY[One-line summary: counts per axis + worst issue]
    
    NOTE["⚠️ NEVER merge or rerank findings across axes"]
    SUMMARY --> DONE([Done])
```

## writing-fragments — fragment accumulator

```mermaid
flowchart TD
    START([/writing-fragments]) --> INITIAL_PROMPT[Capture fragments from INITIAL user prompt immediately]
    INITIAL_PROMPT --> PATH{File path provided?}
    PATH -- no --> ASK_PATH[Ask once for save path]
    PATH -- yes --> FIRST_WRITE
    ASK_PATH --> FIRST_WRITE

    FIRST_WRITE[First write: H1 working title ONLY] --> LOOP

    LOOP[Interview user relentlessly - no phases, no structure]
    LOOP --> FRAGMENT_EMERGES{Fragment emerges from either side?}
    FRAGMENT_EMERGES -- yes --> REREAD[Re-read file from disk - preserve user edits]
    REREAD --> APPEND["Append: \n---\n + fragment text"]
    APPEND --> MENTION["Mention in passing: 'adding that'"]
    MENTION --> LOOP

    FRAGMENT_EMERGES -- no --> EDITING_CMD{User editing command?}
    EDITING_CMD -- "cut last" --> CUT_FRAG[Remove last fragment]
    EDITING_CMD -- "rewrite that one" --> REWRITE_FRAG[Edit fragment in place]
    EDITING_CMD -- "merge those two" --> MERGE_FRAG[Merge fragments]
    CUT_FRAG & REWRITE_FRAG & MERGE_FRAG --> LOOP
    EDITING_CMD -- none --> LOOP

    LOOP -- user ends session --> DONE([Done - pile not necessarily empty])

    subgraph FORMAT["Fragment file format"]
        F1["# Working title"]
        F2["fragment-1 content"]
        F3["---"]
        F4["fragment-2 content"]
        F5["No headings inside body, no tags, no imposed order"]
    end
```

## writing-beats — beat-by-beat journey

```mermaid
flowchart TD
    START([/writing-beats rawMaterial]) --> PATH{Article path provided?}
    PATH -- no --> ASK_PATH[Ask once]
    PATH -- yes --> BEAT_0

    ASK_PATH --> BEAT_0

    BEAT_0{First beat?}
    BEAT_0 -- yes --> DRAFT_STARTS[Draft 2-3 candidate starting beats from raw material]
    DRAFT_STARTS --> PREVIEW[Preview where each might lead]
    PREVIEW --> USER_PICKS[User picks starting beat]
    USER_PICKS --> WRITE_BEAT

    BEAT_0 -- no --> REREAD[Re-read article file from disk]
    REREAD --> OFFER_NEXT[Offer 2-3 candidate next beats from current article position]
    OFFER_NEXT --> USER_PICKS2[User picks next beat]
    USER_PICKS2 --> WRITE_BEAT

    WRITE_BEAT[Write ONLY that one beat to article file]
    WRITE_BEAT --> INCREMENT[beat_count++]
    INCREMENT --> NATURAL_END{Natural end of journey?}
    NATURAL_END -- yes --> DONE([Done])
    NATURAL_END -- no --> BEAT_0

    EDIT_CMD{User editing command?}
    EDIT_CMD -- "rewrite that beat" --> EDIT_INPLACE[Edit beat in place, leave rest alone]
    EDIT_CMD -- "go back to beat N" --> EDIT_PREV[Edit beat N in place, restart from there]
    EDIT_INPLACE & EDIT_PREV --> BEAT_0

    NOTE["End = natural end of journey\nNOT = pile is empty\nLeftover fragments are expected and fine"]
```

## writing-shape — article shaping

```mermaid
flowchart TD
    START([/writing-shape rawMaterial]) --> READ_PILE[Read input pile in full]
    READ_PILE --> PATH{Article path provided?}
    PATH -- no --> ASK_PATH[Ask once]
    PATH -- yes --> OPENINGS
    ASK_PATH --> OPENINGS

    OPENINGS[Draft 2-3 candidate openings - different thesis/angle]
    OPENINGS --> USER_PICK[User picks or composes hybrid]
    USER_PICK --> OPENING_DEFINES[Chosen opening defines what article must do]

    OPENING_DEFINES --> GROWTH_LOOP

    subgraph GROWTH_LOOP["Growth loop"]
        GL1["Ask: what does reader need to hear next?"]
        GL2[Pull material from pile]
        GL3[Argue about format with user]
        GL3a[prose vs list]
        GL3b[inline vs callout]
        GL3c[table vs repeated structure]
        GL3d[quote vs paraphrase]
        GL3e[code block vs inline code]
        GL4[Agree on format]
        GL5[Re-read article from disk - preserve user edits]
        GL6[Append agreed block to article file]
        GL7{Article done?}
        GL8[User decides done]
        GL1 --> GL2 --> GL3
        GL3 --> GL3a & GL3b & GL3c & GL3d & GL3e
        GL3a & GL3b & GL3c & GL3d & GL3e --> GL4 --> GL5 --> GL6 --> GL7
        GL7 -- no --> GL1
        GL7 -- yes --> GL8
    end

    GL8 --> DONE([Done])

    GAP{Pile lacks needed content?}
    GAP -- yes --> NAME_GAP["Name gap explicitly: 'pile doesn't have X — give me one or we cut'"]
    NAME_GAP --> FILL_OR_CUT{User fills gap or cuts section}

    RULES["Rules:\n- Raw material = READ-ONLY\n- Article = append per block\n- Edit in-place only by user request\n- Push back on weak transitions\n- If paragraph doesn't earn its place: cut it"]
```
