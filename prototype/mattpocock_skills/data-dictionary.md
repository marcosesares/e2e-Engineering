### TriageState
🟢 CONFIRMED — issue state machine in `triage`

**State roles** (exactly one per issue):

| State | Meaning |
|-------|---------|
| `needs-triage` | Maintainer needs to evaluate |
| `needs-info` | Waiting on reporter for more information |
| `ready-for-agent` | Fully specified, AFK agent can pick up |
| `ready-for-human` | Needs human implementation (judgment calls, external access, etc.) |
| `wontfix` | Will not be actioned |

**Category roles** (exactly one per issue):

| Category | Meaning |
|---------|---------|
| `bug` | Something is broken |
| `enhancement` | New feature or improvement |

**Invariant**: Every triaged issue carries exactly 1 state role + 1 category role.

### DependencyCategory
🟢 CONFIRMED — taxonomy from `improve-codebase-architecture/DEEPENING.md`

| Category | Characteristics | Testing strategy |
|---------|----------------|-----------------|
| `in-process` | Pure computation, no I/O | Merge modules; test directly |
| `local-substitutable` | Has local stand-in (PGLite, in-memory fs) | Use stand-in in test suite; seam is internal |
| `remote-owned` | Your own service across network | Port + adapter; in-memory adapter for tests |
| `true-external` | Third-party (Stripe, Twilio, etc.) | Inject port; mock adapter for tests |
