## 1. Resumo executivo

The `productivity/` bucket delivers 4 AI agent skills for non-code workflow support: token-compression mode (`caveman`), standalone requirements elicitation (`grill-me`), agent-to-agent context transfer (`handoff`), and skill authoring scaffolding (`write-a-skill`). All four are published in `plugin.json`. They operate independently of the engineering hard-dependency stack and require no per-repo setup.

## 4. Regras de negócio

1. **RN-01:** Caveman mode persists across ALL turns once triggered — never auto-deactivates. 🟢
   - Origem: `state-machines.md#2-caveman-communication-mode`

2. **RN-02:** Caveman exception sub-state is temporary; COMPRESSING resumes immediately after the clear section. 🟢
   - Origem: `state-machines.md#invariants`

3. **RN-03:** Code blocks and exact error quotes are reproduced verbatim regardless of caveman state. 🟢
   - Origem: `state-machines.md#invariants`

4. **RN-04:** `grill-me` asks one question at a time; no maximum cap. 🟢
   - Origem: `domain.md#grilling-rules`

5. **RN-05:** `handoff` generates a structured document — not a passthrough of session history. 🟢
   - Origem: `adrs/0005-handoff-is-not-passthrough.md`

6. **RN-06:** `write-a-skill` must update `plugin.json` AND `README.md` for skills in `engineering/` or `productivity/`. 🟢
   - Origem: `CLAUDE.md`

7. **RN-07:** New skill descriptions generated by `write-a-skill` must be ≤ 1024 characters and include "Use when [trigger]". 🟢
   - Origem: `domain.md#skill-authoring-rules`

## 7. Critérios de Aceitação

```gherkin
Cenário: caveman — persistent activation
  Dado o developer triggers caveman mode with "be brief"
  Quando the developer sends the next 5 messages
  Então all 5 responses are compressed
    E caveman mode has not auto-deactivated

Cenário: caveman — exception sub-state for irreversible action
  Dado caveman mode is ACTIVE
  Quando the agent is about to execute an irreversible operation (e.g., git reset --hard)
  Então the agent enters EXCEPTION sub-state and uses full verbosity for that section
    E COMPRESSING mode resumes on the next turn

Cenário: caveman — code blocks verbatim
  Dado caveman mode is ACTIVE
  Quando the agent produces a response containing a code block
  Então the code block is reproduced verbatim without compression

Cenário: grill-me — one question per turn
  Dado a developer invokes /grill-me with a vague feature request
  Quando the skill begins elicitation
  Então each agent turn contains exactly one question
    E the session continues until the spec is complete

Cenário: handoff — self-contained document
  Dado a developer invokes /handoff mid-task
  Quando a fresh agent reads the handoff document in a new session
  Então the fresh agent can continue the task without reading the prior conversation

Cenário: write-a-skill — registry updates for engineering skill
  Dado the developer asks write-a-skill to scaffold a new engineering/ skill
  Quando the skill is created
  Então plugin.json contains an entry for the new skill
    E top-level README.md contains a linked entry for the new skill
    E skills/engineering/README.md contains a linked entry
```
