# User Stories: Writing Workflows

> Generated by Reversa Writer on 2026-05-15
> Covers: in-progress/ writing skills (writing-beats, writing-fragments, writing-shape)
> Also covers: productivity/ handoff skill for session continuity
> Confidência: 🟢 CONFIRMADO, 🟡 INFERIDO

## Epic: Idea Capture

### US-01 — Capture raw ideas before they're lost (🟢)
**As a** writer with fragmented thoughts about an article,
**I want** to be interviewed to surface and capture those thoughts as fragments,
**So that** my best ideas are preserved before the session ends, even if I don't have a complete article structure yet.

**Acceptance criteria**:
- [ ] My first message is captured as a fragment before the interview begins
- [ ] The interview surfaces one idea at a time via single questions
- [ ] Each surfaced fragment is appended to a file on disk
- [ ] My manual edits to the fragment file between turns are preserved
- [ ] The session ends when I signal done — not when ideas are "exhausted"
- [ ] The pile is intentionally left incomplete — that is expected

### US-02 — Edit collected fragments mid-session (🟢)
**As a** writer reviewing my accumulated fragments,
**I want** to cut, rewrite, or merge fragments during the session,
**So that** the pile stays clean as I go, not just at the end.

**Acceptance criteria**:
- [ ] I can say "cut the last fragment" and it is removed
- [ ] I can say "rewrite fragment 3" and the skill prompts me for the new version
- [ ] I can say "merge fragments 1 and 2" and they are combined
- [ ] After each editing command: the skill re-reads the file from disk before applying the edit

## Epic: Article Development

### US-03 — Develop an article beat by beat (🟢)
**As a** writer with an article in progress,
**I want** to grow the article one beat at a time, choosing from candidates the AI offers,
**So that** I maintain creative control over the article's direction while the AI handles the writing.

**Acceptance criteria**:
- [ ] The AI presents 2-3 beat candidates before writing anything
- [ ] I choose which beat to write
- [ ] The AI writes exactly one beat and stops
- [ ] My manual edits to the article between turns are preserved (disk re-read before each write)
- [ ] After each beat, 2-3 next-beat candidates are offered
- [ ] The session ends at the natural narrative end — not when candidate ideas run out

### US-04 — Rewrite a beat I don't like (🟢)
**As a** writer unhappy with a written beat,
**I want** to go back and try a different beat,
**So that** I'm not stuck with the AI's first attempt.

**Acceptance criteria**:
- [ ] I can request a rewrite of the last beat
- [ ] The skill re-presents beat candidates for that position
- [ ] I pick a new candidate and it replaces the previous beat
- [ ] The article file reflects the replacement

## Epic: Article Shaping

### US-05 — Shape raw material into a structured article (🟢)
**As a** writer with a document full of raw notes, quotes, and ideas,
**I want** the AI to help me shape that material into a structured article,
**So that** I have a publishable draft without having to manually reorganise everything.

**Acceptance criteria**:
- [ ] The raw material file is never modified — it is read-only throughout the session
- [ ] The shaped article is written to a separate target file
- [ ] My edits to the shaped article between turns are preserved (disk re-read before each write)
- [ ] The AI proposes the structure incrementally and I can redirect at each step
- [ ] After the session: raw material file is byte-for-byte identical to before

### US-06 — Continue shaping from a previous session (🟡)
**As a** writer returning to a shaping session after a break,
**I want** to pick up where I left off,
**So that** I don't have to start the shaped article over.

**Acceptance criteria**:
- [ ] The skill reads the existing shaped article content at session start
- [ ] Shaping continues from the current state of the shaped file
- [ ] Existing shaped content is not overwritten

## Epic: Session Continuity

### US-07 — Hand off a writing session to a fresh agent (🟢)
**As a** writer whose session is getting long,
**I want** to generate a handoff document before starting a new session,
**So that** the fresh agent can continue exactly where I left off without me re-explaining everything.

**Acceptance criteria**:
- [ ] Handoff document contains: current article state, next beat or action, open questions, relevant file paths
- [ ] A fresh agent reading only the handoff document can continue the session
- [ ] The handoff document is saved to a file I can reference in the new session

## Epic: Code Review

### US-08 — Get a two-axis code review (🟢)
**As a** developer finishing a feature,
**I want** a review that covers both whether the code works AND whether it's well-written,
**So that** I catch both functional bugs and craft issues before merging.

**Acceptance criteria**:
- [ ] Review output has a Correctness section (does it work? bugs? edge cases? test adequacy?)
- [ ] Review output has a Craft section (readable? well-named? follows patterns?)
- [ ] Findings are classified: critical (must fix) vs suggested (improve)
- [ ] I receive a confidence marker (🟢/🟡/🔴) on the overall review

### US-09 — Review multiple files efficiently (🟡)
**As a** developer with a multi-file PR,
**I want** all files reviewed concurrently rather than one at a time,
**So that** the review completes faster on large changesets.

**Acceptance criteria**:
- [ ] Files are reviewed in parallel (when runtime supports it)
- [ ] Output is the same structured ReviewReport regardless of whether parallel or sequential
- [ ] Sequential fallback does not produce an error — it just takes longer
