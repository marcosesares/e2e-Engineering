## High-Impact Change Scenarios

### Scenario A: `triage-labels.md` schema changes
**Impacted**: `triage`, `to-issues`, `to-prd` — all three have hard dependency on this file. A schema change (e.g., renaming label keys) breaks all three simultaneously.
**Risk**: 🔴 HIGH
**Mitigation**: ADR-0001 documents the hard/soft split. Any schema change requires updating setup-matt-pocock-skills to produce the new schema.

### Scenario B: `setup-matt-pocock-skills` changes seeded config format
**Impacted**: `triage`, `to-issues`, `to-prd` immediately. `diagnose`, `tdd`, `improve-arch` affected if CONTEXT.md or ADR structure changes.
**Risk**: 🔴 HIGH (hard dependencies) / 🟡 MEDIUM (soft dependencies)
**Mitigation**: Idempotency rule means re-running setup won't overwrite existing config — existing repos stay on old schema until manually updated.

### Scenario C: `CLAUDE.md` governance rules change
**Impacted**: All skills — CLAUDE.md defines structural invariants (100-line limit, description format, publication rules) that all skills must comply with.
**Risk**: 🔴 HIGH (structural), but changes require active contributor effort to propagate.

### Scenario D: `write-a-skill` changes scaffolding template
**Impacted**: `plugin.json`, `link-skills.sh`, `README.md` — new skills created with the new template will have different structure. Existing skills unaffected.
**Risk**: 🟡 MEDIUM (forward-only impact).

### Scenario E: Issue tracker API changes (GitHub/GitLab)
**Impacted**: `triage`, `to-issues`, `to-prd` — all use natural-language API interaction.
**Risk**: 🟡 MEDIUM — skills describe intent ("apply label X"), not raw API calls. Claude Code adapts to API changes within a generation.
