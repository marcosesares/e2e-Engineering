## System Summary

The **Matt Pocock Skills Library** is a content-delivery system for AI agent instruction sets ("skills"). It is not a traditional software application — it has no web server, no database, no runtime services. Its "output" is a set of Markdown files that program AI agents (primarily Claude Code) to execute specialized workflows.

| Property | Value |
|----------|-------|
| Primary artifact | `SKILL.md` files — Markdown natural-language programs |
| Runtime | AI agent (Claude Code, Codex, or compatible) |
| Distribution | `npx skills@latest` or symlink via `link-skills.sh` |
| Persistence layer | Filesystem (user's project repo) |
| Concurrency model | N/A — skills are invoked sequentially per agent session |
| Deployment infrastructure | None — pure content distribution |

## Key Architectural Properties

### Content-as-Code
Skills are natural-language programs. The "compiler" is the AI agent that reads them. Quality = instruction clarity + invariant precision + example richness. There is no type system, no linter, no test runner enforcing correctness.

### Tracker-Agnostic Design 🟢
Issue-tracker operations are abstracted behind a mapping layer (`triage-labels.md`) since commit `7afa86d`. Skills that interact with issue trackers (to-issues, to-prd, triage) accept GitHub, GitLab, and local-markdown as equivalent backends.

### Hard/Soft Dependency Split 🟢
Skills are classified by whether they require per-repo configuration (`/setup-matt-pocock-skills`):
- **Hard** (to-issues, to-prd, triage): broken without config
- **Soft** (diagnose, tdd, improve-codebase-architecture, zoom-out): degraded but functional without config

### Delete-or-Absorb Lifecycle 🟢
Prototypes and deprecated artifacts do not persist. Prototypes: deleted or promoted to production. Deprecated skills: moved to `deprecated/`, excluded from registry.

### Publication Boundary 🟢
Not all skills are published. Only `engineering/` and `productivity/` appear in `plugin.json` and are linked into `~/.claude/skills/`. `misc/` exists in the repo but is unlisted. `deprecated/` and `personal/` are excluded.

## Technical Debts

🔴 **No automated governance**: no CI check verifies that all engineering/productivity skills appear in `plugin.json` and `README.md`. Manual discipline only.

🔴 **No skill versioning**: no `version` field in SKILL.md frontmatter. No mechanism to pin specific skill versions.

🔴 **No regression test suite**: skills are natural language — no automated test verifies behavior after edits.

🔴 **in-progress graduation criteria undefined**: no documented threshold for when an in-progress skill graduates to an active bucket.

🟡 **SKILL.md 100-line guideline**: complex skills (grill-with-docs, triage, tdd) risk exceeding the guideline as functionality grows. No automated enforcement.

🟡 **Supporting file proliferation**: `tdd/` has 4 supporting files; as skills grow in complexity, the number of reads an agent must do on invocation grows, increasing latency.

## Cross-cutting Concerns

| Concern | Implementation | Coverage |
|---------|---------------|----------|
| Language/output format | `doc_language` / `chat_language` per `state.json` | All skills |
| AI comment tagging | `> *This was generated by AI during triage.*` prefix | Triage only |
| Codebase-first (explore before ask) | Invariant in grill-with-docs, triage | Grilling skills |
| Disk re-read before write | Invariant in all writing-* skills | Writing skills only |
| Token efficiency | `caveman` mode, 100-line SKILL.md guideline | All skills |
| Backwards compatibility | No shims — breaking changes accepted (ADRs 0002, 0012, 0013) | All skills |
