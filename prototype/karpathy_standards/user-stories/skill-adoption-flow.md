## US-01 — Install as Plugin (Cross-Project)

**As** a developer who uses Claude Code across multiple projects,
**I want** to install the Karpathy guidelines as a plugin once,
**So that** all four behavioral principles are active in every project without copying files.

**Acceptance criteria:**
```gherkin
Given I have Claude Code installed
When I run `/plugin marketplace add forrestchang/andrej-karpathy-skills`
And run `/plugin install andrej-karpathy-skills@karpathy-skills`
Then the skill is active in my current session
And I can verify by asking Claude to "list the four Karpathy guidelines"
```

**Priority:** Must · **Confidence:** 🟢

## US-02 — Install via CLAUDE.md (Per-Project)

**As** a developer working on a project with an existing `CLAUDE.md`,
**I want** to append the skill content to my project's `CLAUDE.md`,
**So that** any collaborator who opens the project gets the guidelines automatically without installing a plugin.

**Acceptance criteria:**
```gherkin
Given a project has an existing CLAUDE.md
When I append the content of SKILL.md to it
Then any Claude Code session in that project applies all four principles
And no plugin installation is required for collaborators
```

**Priority:** Must · **Confidence:** 🟢

## US-03 — Verify Skill Is Active

**As** a developer who has installed the skill,
**I want** to confirm the guidelines are loaded in my current session,
**So that** I am not coding with an inactive skill without knowing.

**Acceptance criteria:**
```gherkin
Given the skill is installed (either path)
When I ask Claude "What are the Karpathy guidelines?"
Then Claude lists all four principles by name with their key rules
And does not say it has no knowledge of them
```

**Priority:** Should · **Confidence:** 🟡 — no formal activation acknowledgment mechanism defined

## US-04 — Apply Think Before Coding on an Ambiguous Request

**As** a developer sending an ambiguous implementation request,
**I want** Claude to surface my assumptions before writing code,
**So that** I catch misunderstandings before they become wrong implementations.

**Acceptance criteria:**
```gherkin
Given the skill is active
And I send "add caching to the API"
When Claude applies Think Before Coding
Then Claude asks which cache layer (in-memory vs. Redis vs. CDN) before writing code
```

**Priority:** Must · **Confidence:** 🟢

## US-05 — Apply Simplicity First on a Code Generation Request

**As** a developer asking for a new feature,
**I want** Claude to generate the minimum viable implementation,
**So that** the output is easy to review and does not include unrequested scope.

**Acceptance criteria:**
```gherkin
Given the skill is active
And I ask "add a greet() function that returns Hello, <name>"
When Claude applies Simplicity First
Then the output is a single 1–3 line function with no config, no logging, no validation
```

**Priority:** Must · **Confidence:** 🟢

## US-06 — Apply Surgical Changes on an Edit Request

**As** a developer asking Claude to fix a specific bug,
**I want** the diff to touch only the lines directly required by the fix,
**So that** code review is fast and the risk of unintended side effects is minimal.

**Acceptance criteria:**
```gherkin
Given the skill is active
And I ask "fix the null check in processOrder()"
When Claude applies Surgical Changes
Then the diff touches only the null check logic inside processOrder()
And no other functions, comments, or imports are changed
```

**Priority:** Must · **Confidence:** 🟢

## US-07 — Apply Goal-Driven Execution on a Multi-Step Task

**As** a developer assigning a complex multi-step task,
**I want** Claude to define success criteria and a plan before starting,
**So that** I can review the approach and Claude can execute autonomously.

**Acceptance criteria:**
```gherkin
Given the skill is active
And I ask "migrate the users table to the new schema"
When Claude applies Goal-Driven Execution
Then Claude produces a step-by-step plan with verify checks before writing any code
And each step includes an explicit "→ verify:" condition
```

**Priority:** Must · **Confidence:** 🟢
