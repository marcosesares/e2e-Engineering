## Skill Activation & Execution Flow

This flowchart shows how the skill is activated and how its 4 principles interact at runtime.

```mermaid
flowchart TD
    A([User installs skill]) --> B{Install method?}
    B -->|Plugin| C["/plugin install andrej-karpathy-skills@karpathy-skills"]
    B -->|CLAUDE.md| D["Copy/append CLAUDE.md to project"]
    C --> E[All 4 principles active in LLM context]
    D --> E

    E --> F([User sends request to LLM])

    F --> G{Is task trivial?}
    G -->|Yes — simple typo, one-liner| H[Apply judgment, skip full rigor]
    G -->|No| I[Apply Think Before Coding]

    I --> J{Assumptions clear?}
    J -->|No — uncertain| K[Ask for clarification]
    K --> F
    J -->|Multiple interpretations| L[Present all interpretations]
    L --> M[Wait for user to pick]
    M --> I
    J -->|Simpler approach exists| N[Say so, push back]
    N --> F
    J -->|Yes — clear| O[Apply Goal-Driven Execution]

    O --> P[Transform task to verifiable goal]
    P --> Q{Multi-step task?}
    Q -->|Yes| R[State plan: Step → verify check]
    Q -->|No| S[Define single success criterion]
    R --> T[Implement step 1]
    S --> T

    T --> U{Editing existing code?}
    U -->|Yes| V[Apply Surgical Changes]
    U -->|No — new code| W[Apply Simplicity First]

    V --> X[Only touch lines tracing to request]
    X --> Y[Match existing style]
    Y --> Z{Own changes created orphans?}
    Z -->|Yes| AA[Remove unused imports/vars from OWN changes]
    Z -->|No| AB[Do NOT touch pre-existing dead code]
    AA --> AC[Verify step passes]
    AB --> AC

    W --> AD{Would senior engineer call this overcomplicated?}
    AD -->|Yes| AE[Simplify — rewrite to fewer lines]
    AE --> AD
    AD -->|No| AC

    AC --> AF{More steps in plan?}
    AF -->|Yes| T
    AF -->|No| AG([Done — all verifications passed])
```

## Notes

- 🟢 **CONFIRMADO** — All decision nodes derive directly from explicit rules in `SKILL.md`.
- 🟡 **INFERIDO** — The "trivial task" branch has no formal criteria; the LLM decides.
- The loop between `T → AC → AF → T` represents the Goal-Driven Execution loop.
- "Surgical Changes" and "Simplicity First" are not mutually exclusive but apply to different contexts (edit vs. new code).
