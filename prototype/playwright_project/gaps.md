## 🟡 Moderate Gaps (spec accurate enough to guide work but with wrong details)

### MG-01 — RBAC matrices presume the wrong role set
**Evidência:** `permissions.md` enumerates BidProject / BidPackage / LineItem / Fee / Requirement / Bidder / Settings RBAC tables with 4 roles including `Viewer`. With `Viewer` removed, the Viewer columns of every table become invalid.
**Especs afetadas:** all role tables in `permissions.md`; `UserPermissionsMatrix/design.md`; `architecture.md` RBAC sections (if present).
**Linked to:** Pergunta 3.

### MG-02 — Locality-based authorization is inferred, not in code
**Evidência:** `permissions.md:140-157` claims user `Location` may restrict permissions. Tests only set `Location` as a field; no test asserts locality gating.
**Especs afetadas:** `permissions.md` (Restrições por Localidade), `UserPermissionsMatrix/design.md`.
**Linked to:** Pergunta 10.

### MG-03 — "Bidder" treated as a login role
**Evidência:** `permissions.md:81,194-202` introduces a "Bidder" role for `Requirement.Respond` and "view own responses". Bidder is a data entity (`Models/Bidder.cs`), not a `UserRole`.
**Especs afetadas:** `permissions.md` (conditional rules), domain glossary cross-references.
**Linked to:** Pergunta 9.

### MG-04 — Concrete API endpoints in flows are inferred
**Evidência:** Flows like `Bidders/flows.md:29-37,128-131,177-178,229` state `POST /commands/add-bidder`, `PUT /commands/update-bidder`, `DELETE /commands/delete-bidder` with full request bodies. None of the strings are visible in the test corpus and the spec marks them 🟢.
**Especs afetadas:** every `flows.md` in the corpus that quotes endpoint paths (Bidders certainly; others to be audited).
**Linked to:** Pergunta 12.

### MG-05 — `Bidders/design.md` "BidderModal Lifecycle" state machine is invented

**Especs afetadas:** `Bidders/design.md` (State Machine block).

### MG-06 — Sorting + pagination claims for Bidders grid
**Evidência:** `Bidders/requirements.md:74` ("Sorting by Company Name (ascending default) / Pagination if >10 bidders (10 per page)"), `Bidders/design.md` "Grid display", `Bidders/flows.md` Flow-06 — no evidence in code; no test exercises pagination/sorting.
**Especs afetadas:** `Bidders/requirements.md`, `Bidders/design.md`, `Bidders/flows.md`, `Bidders/tasks.md` (B-007).
