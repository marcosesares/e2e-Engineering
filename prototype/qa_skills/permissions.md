# Permissions Matrix — BeckTech.QA.Tools

> Generated by Reversa Detective · 2026-05-23
> Confidence: 🟢 CONFIRMADO | 🟡 INFERIDO | 🔴 LACUNA

This system is a **developer tooling framework**, not a multi-user RBAC application. "Roles" are defined by the position of a machine in the distribution topology, and by the tool permissions declared in agent frontmatter.

---

## 1. Distribution Topology Roles

| Role | Who | Description |
|------|-----|-------------|
| **Maintainer** | BeckTech QA Tools team member with write access to `BeckTech.QA.Tools` | Can edit `claude-skills/`, publish `published-skills`, and update pipeline YAML. |
| **Consumer** | QA engineer in a downstream repo (e.g., `BeckTech.QA.Estimator`) | Receives skills via git subtree. Can invoke skills; cannot edit shared files. |
| **CI/CD Pipeline** | Azure DevOps Build Service | Runs `publish-claude-skills.yml` and `publish-testkit.yml`; pushes to `published-skills` and NuGet feed. |

## 2. File Modification Rights

| Target | Maintainer | Consumer | CI/CD Pipeline | Guard Enforces |
|--------|-----------|----------|----------------|----------------|
| `claude-skills/**` (source) | ✅ Full | ❌ Not in their repo | ❌ Read-only | N/A |
| `.claude/**` (synced, shared) | ✅ Via source | ❌ Blocked by hook | ❌ | ✅ `guard-shared-skills.ps1` |
| `.claude/skills/repo-context/**` | ✅ | ✅ Allowed (local) | ❌ | ✅ Not in SHARED_MANIFEST |
| `CLAUDE.md` | ✅ Via template | ✅ Allowed (generated locally) | ❌ | ❌ (not in manifest) |
| `.claude/settings.json` | ✅ Via skill | ✅ Allowed (via setup skills) | ❌ | ❌ (not in manifest) |
| `~/.claude/settings.json` | ✅ | ✅ (user-scope) | ❌ | ❌ |
| `published-skills` branch | ❌ Direct | ❌ | ✅ Force-push | CI gate (pr: none) |
| DESTINI-Web NuGet feed | ❌ Direct | ❌ | ✅ Push | `--skip-duplicate` |

**Bypass:** `QA_ALLOW_SHARED_EDIT=1` — overrides the guard for Maintainer use. 🟢

## 3. Agent Tool Permissions

| Agent | Allowed Tools | Prohibited | Rationale |
|-------|--------------|------------|-----------|
| `qa-investigate` | `Read`, `Glob`, `Grep`, `Bash` | `Write`, `Edit`, `Agent` | Read-only investigation. Cannot modify project files. Writes only `investigation.md` (output contract). | 🟢 |

## 4. Skill Invocability

Skills have a `user-invocable` flag in frontmatter. Determines whether the skill appears in Claude Code's skill list and can be directly invoked by the user.

| Category | Examples | user-invocable | Invoked By |
|----------|----------|---------------|------------|
| **Workflow entry points** | `work-on`, `fix-qa-bug`, `implement-story`, `apply-test-plan` | ✅ | User command (`/work-on QA-1234`) |
| **Planning skills** | `plan-feature-coverage`, `plan-regression-coverage`, `test-plan` | ✅ | User command |
| **Change management** | `plan-change`, `review-change`, `commit-change` | ✅ | User command |
| **ADO mechanics** | `create-ado-test-cases`, `update-ado-test-cases`, `manage-ado-test-suite` | ✅ | User or `apply-test-plan` |
| **Jira** | `fetch-jira-item` | ✅ (auto + manual) | `work-on` or user |
| **Setup** | `onboard`, `setup-*`, `refresh-setup` | ✅ | User command |
| **Principles** | `principles` | ❌ | Agent frontmatter `skills:` list |

## 5. External System Permissions Required

### Jira / Atlassian

| Operation | Credentials Required | Script / Skill |
|-----------|---------------------|---------------|
| Read issue | `ATLASSIAN_EMAIL` + `ATLASSIAN_API_TOKEN` (REST preferred) or Atlassian MCP OAuth | `fetch-jira-issue.ps1`, `fetch-jira-items-batch.ps1` |
| Add comment | `-Pat` param (explicit) or Atlassian MCP | `add-jira-comment.ps1` |
| Update fields | `ATLASSIAN_EMAIL` + `ATLASSIAN_API_TOKEN` or `-Pat` | `update-jira-issue.ps1` |
| Transition status | `-Pat` param (explicit) | `transition-jira-issue.ps1` |
| Set custom field (test plan IDs) | `ATLASSIAN_EMAIL` + `ATLASSIAN_API_TOKEN` | `set-jira-test-plan-ids.ps1` |

**Gap:** Three Jira scripts (`query-jira.ps1`, `add-jira-comment.ps1`, `transition-jira-issue.ps1`) require `-Pat` explicitly and do not support env-var fallback — inconsistent with the other four scripts. 🟢 (known bug)

### Azure DevOps

| Operation | Credentials Required | Scope |
|-----------|---------------------|-------|
| Create/update work items | `ADO_PAT` | `Test Management Read+Write`, `Work Items Read` |
| Read test plans/suites | `ADO_PAT` | `Test Management Read` |
| Create PRs | `ADO_PAT` | `Code Write` 🔴 (exact scope not documented) |
| Update PR reviewers/votes | `ADO_PAT` | `Code Write` 🔴 |
| Publish NuGet (CI) | Azure Build Service (implicit service connection) | DESTINI-Web feed write 🔴 |
