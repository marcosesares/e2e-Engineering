## Pergunta 20 — Agent Read-Only Constraint ✅ Respondida

**Resposta:** Read-only is enforced by **two layers**, both partial:

1. **Tool whitelist (frontmatter):** `claude-skills/agents/qa-investigate.md` line 4 declares `tools: Read, Glob, Grep, Bash`. **No `Write`/`Edit`/`NotebookEdit` tool granted.** That blocks the obvious mutation tools.
2. **Prompt policy:** Lines 14-19 of the same file say *"Read-only. You MUST NOT edit, write, or delete any project files. Write tool is ONLY for writing your investigation brief to the output file."* But the **frontmatter does not grant Write**, so the "Write tool ONLY for ..." sentence is **internally contradictory** with the tools list.

**Escape hatch:** `Bash` is granted. The agent can mutate files via shell redirects (`echo > file`, `cp`, `mv`, `rm`). Read-only is therefore **not enforceable** through tool granting alone; it relies on agent self-discipline plus the absence of Write/Edit tools.

The contradiction in `qa-investigate.md` (prompt cites Write tool that isn't granted) is a separate **defect in the agent definition** — investigation.md must be writable somehow. Practical mechanism: orchestrator passes the output path; the agent uses `Bash` (`Out-File` / `Set-Content`) to write it. Will add this to `agents/design.md` as a clarification and flag the prompt/frontmatter contradiction in `agents/edge-cases.md`.
