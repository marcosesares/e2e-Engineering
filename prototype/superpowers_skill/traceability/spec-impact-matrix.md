## Component Dependency Matrix (Detailed)

### **1. Skill Execution Engine Dependencies**

| Component | Depends On | Dependency Type | Impact If Failed |
|-----------|---|---|---|
| **Trigger Matcher** | Bootstrap Loader | direct call | Cannot initialize trigger rules → no auto-invocation |
| **Trigger Matcher** | Skill Loader | data (skill list) | Cannot load matched skill → workflow stalls |
| **Skill Loader** | Git Repo | data read | Cannot locate SKILL.md → fatal error |
| **Skill Parser** | Skill Loader | direct call | Cannot parse metadata/phases → skill unusable |
| **Context Injector** | Session Manager | data read | Cannot gather session context → missing info for skill |
| **Gate Enforcer** | Skill Parser | data (gate defs) | Cannot enforce gates → process violations allowed |
| **Red Flag Detector** | Skill Parser | data (red flag list) | Cannot match patterns → anti-patterns not caught |
| **Skill Renderer** | Skill Parser, Context Injector | direct call | Cannot present workflow → agent has no guidance |
| **Checkpoint Saver** | Session Manager | direct call | Cannot persist state → cannot resume later |

### **2. Orchestration Dependencies**

| Component | Depends On | Dependency Type | Impact If Failed |
|-----------|---|---|---|
| **Worktree Manager** | Tool Abstraction Layer | direct call (git worktree add) | Cannot isolate workspace → branch pollution |
| **Agent Dispatch** | Worktree Manager | ordering | Agents spawned in non-isolated dir → cross-contamination |
| **Agent Dispatch** | Tool Abstraction Layer | direct call (spawn agent) | Cannot create subagents → parallel work impossible |
| **Review Loop** | Agent Dispatch | orchestration | Cannot run spec/quality reviews → poor code quality |

### **3. Harness Integration Dependencies**

| Component | Depends On | Dependency Type | Impact If Failed |
|-----------|---|---|---|
| **Bootstrap Loader** | Harness Manifest | configuration read | Cannot load bootstrap → no auto-trigger |
| **Tool Abstraction Layer** | Harness Manifest | configuration read | Tools unavailable or wrong API → commands fail |
| **Tool Abstraction Layer** | External Tools (git, bash, file I/O) | direct call | Cannot execute → workflow stalled |
| **Skill Loader** | Harness Manifest (skills_search_paths) | configuration read | Cannot find skills → skills unavailable |

## Cross-Component Impact Chains

### **Chain 1: User Intent → Skill Execution**

```
Bootstrap Loader (harness loads)
  ↓ registers
Trigger Matcher (initializes trigger rules)
  ↓ on user message
Trigger Matcher → Intent Extractor (extract user intent)
  ↓ match against rules
Trigger Matcher → Skill Loader (load matched skill from repo)
  ↓ parse
Skill Parser (extract metadata, phases, gates, flags)
  ↓ inject context
Context Injector (gather codebase, session, user context)
  ↓ check gates
Gate Enforcer (evaluate hard gates)
  ↓ render if OK
Skill Renderer (format workflow for readability)
  ↓ agent follows
Agent reads skill, executes phase, requests tools
  ↓ tools execute
Tool Abstraction Layer (git, bash, file I/O)
  ↓ saves state
Checkpoint Saver (persist phase completion)

IMPACT: If any component fails:
- Trigger Matcher fails → no skill loading
- Skill Loader fails → cannot proceed
- Gate Enforcer fails → gates bypassed
- Tool Abstraction Layer fails → cannot execute
- Checkpoint Saver fails → cannot resume
```

### **Chain 2: TDD Cycle**

```
trigger: "write code"
  ↓ TRIGGER_MATCHER
"test-driven-development" skill loaded
  ↓ SKILL_PARSER
Metadata + 4 phases + gates + red flags extracted
  ↓ CONTEXT_INJECTOR
Codebase context (language, test framework) injected
  ↓ GATE_ENFORCER
TDD RED gate checked: "test must fail first"
  ↓ (if OK) SKILL_RENDERER
Phase 1 (RED) rendered to agent
  ↓ AGENT EXECUTION
Agent writes test, runs it (expects failure)
  ↓ TOOL_ABSTRACTION_LAYER
Test framework invoked (jest, pytest, etc.)
  ↓ RED_FLAG_DETECTOR
If agent says "I'll test after" → flag matched
  ↓ RED_FLAG_HANDLER
Reality check shown: "Tests after = what does?"
  ↓ (agent self-corrects or overrides) CHECKPOINT_SAVER
Phase 1 completion saved
  ↓ PHASE_ADVANCE
Phase 2 (GREEN) begins

IMPACT TREE (any failure blocks cycle):
  Trigger Match → Skill Load → Parse → Context → Gate → Render → Execute → Flag Check → Checkpoint → Advance
  If any link breaks: cycle stalls
```

### **Chain 3: Subagent-Driven Development (SDD)**

```
Plan ready (writing-plans complete)
  ↓ AGENT_DISPATCH (implementer)
Fresh subagent spawned with task text
  ↓ WORKTREE_MANAGER
Isolated workspace created (git worktree add)
  ↓ TOOL_ABSTRACTION_LAYER (in subagent context)
Agent: read spec, write code, write tests, commit
  ↓ AGENT_DISPATCH (spec_reviewer)
Second subagent spawned
  ↓ SKILL_ENGINE (in reviewer context)
Spec Reviewer reads: code_changes, spec, recent_commits
  ├─ Code matches spec? YES → step 3
  └─ NO → Agent Dispatch implementer again (loop)
  ↓ AGENT_DISPATCH (quality_reviewer)
Third subagent spawned
  ↓ SKILL_ENGINE (in quality context)
Quality Reviewer reads code
  ├─ Approved? YES → task complete ✅
  └─ NO → Agent Dispatch implementer again (loop)

IMPACT: If any component fails:
- Worktree Manager fails → cannot isolate task
- Agent Dispatch fails → cannot spawn reviewers
- Tool Abstraction Layer (in subagent) fails → code cannot be written/tested
- Checkpoint Saver fails → review loops not recorded
```

## Failure Cascades

### **Scenario 1: Skill Loader Fails (Skills Repository Unavailable)**

```
Cause: .claude/skills/ not found or unreadable

Impact Flow:
  Trigger Match → Skill Load ❌
              ↓
  No skill content available
  Workaround: Show cached version (if available)
  Recovery: Admin checks skill repo path, permissions
  
Dependent Workflows Blocked:
  ✗ All skill-based workflows (brainstorming, TDD, debugging, etc.)
  ✗ Agent guidance unavailable
  
Estimated Recovery: 1-5 minutes (admin intervention)
```

### **Scenario 2: Tool Abstraction Layer Fails (Git Unavailable)**

```
Cause: `git` command not in PATH, or harness tool unavailable

Impact Flow:
  Skill Execution → Tool Request ❌
              ↓
  Cannot commit, branch, worktree operations
  
Dependent Workflows Blocked:
  ✗ Worktree creation (finishing-a-development-branch)
  ✗ Git operations (brainstorming context gathering)
  ✗ Checkpoint persistence (if saved to git)
  
Fallback: Bash tool as universal fallback for git
  
Estimated Recovery: Automatic fallback to Bash
```

### **Scenario 3: Gate Enforcer Fails (Hard Gate Bypassed)**

```
Cause: Gate enforcement logic broken or skipped

Impact Flow:
  Gate Enforcer ❌
  TDD RED gate not enforced
  Agent writes code before test
              ↓
  Process violation, test coverage incomplete
  
Long-term Impact:
  ✗ Code quality degradation
  ✗ Harder-to-catch bugs
  ✗ Technical debt accumulation
  
Mitigation: Red Flags still fire (catch most violations)
Recovery: Manual code review, backfill tests
  
Estimated Detection: Next test run (or post-submission review)
```

### **Scenario 4: Checkpoint Saver Fails (State Not Persisted)**

```
Cause: File I/O error, permission denied, disk full

Impact Flow:
  Checkpoint Save ❌
  Session state not persisted
              ↓
  User cannot resume in new session
  Must restart workflow from beginning
  
Affected Operations:
  ✗ `/resume` command returns "no checkpoint found"
  ✗ Long workflows (brainstorming → planning → coding) start over
  
User Experience:
  Loss of context, decisions, partial artifacts
  
Mitigation: In-memory checkpoint (until disk available)
Recovery: Retry checkpoint save, warn user
  
Estimated Impact: High (time lost on long tasks)
```

## Component Criticality Ranking

### **Tier 1: CRITICAL** (System-wide failure if down)
- **Trigger Matcher:** No skill invocation
- **Skill Loader:** No workflow content
- **Tool Abstraction Layer:** No execution capability
- **Bootstrap Loader:** No auto-trigger at session start

### **Tier 2: HIGH** (Workflow failure, workaround possible)
- **Skill Parser:** Skills loaded but not usable
- **Gate Enforcer:** Process gates bypassed (but red flags catch some)
- **Checkpoint Saver:** Cannot resume (but can restart)
- **Worktree Manager:** Cannot isolate (but can warn user)

### **Tier 3: MEDIUM** (Reduced effectiveness, not blockers)
- **Red Flag Detector:** Anti-patterns not caught (but gates still enforced)
- **Context Injector:** Missing context (but agent can continue)
- **Skill Renderer:** Poor presentation (but agent reads markdown)

### **Tier 4: LOW** (Nice-to-have, graceful degradation)
- **Agent Dispatch:** Serial execution fallback available
- **Review Loop:** Manual code review alternative exists

## Integration Testing Checklist

To verify impact chains work correctly:

- [ ] **Trigger → Skill Load:** Message "let's make X" loads brainstorming skill
- [ ] **Skill Parse → Phase Render:** Phases rendered with correct gates & red flags
- [ ] **Gate Enforcement:** TDD RED gate blocks code before test
- [ ] **Red Flag Detection:** "I'll test after" triggers reality check
- [ ] **Checkpoint Persistence:** Completed phase saved, resumable next session
- [ ] **Worktree Creation:** Workspace isolated, no GIT_DIR nesting
- [ ] **Agent Dispatch:** Implementer + spec reviewer + quality reviewer sequence works
- [ ] **Tool Fallback:** Git command falls back to Bash if primary tool unavailable
- [ ] **Multi-Harness:** Same workflows work on Claude Code, Codex, Cursor, OpenCode
