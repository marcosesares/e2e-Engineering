## Interface

This unit has no programmatic API. Its interface is the skill instruction text consumed by an LLM at inference time.

| Install path | Entry point | Consumed by |
|--------------|-------------|-------------|
| Plugin | `skills/karpathy-guidelines/SKILL.md` | Claude Code plugin runtime |
| CLAUDE.md | `CLAUDE.md` (root) | Claude Code direct file read |
| Marketplace | `.claude-plugin/plugin.json` + `marketplace.json` | Claude Code marketplace CLI |

---

## Primary Flow — Plugin Installation

1. User runs `/plugin marketplace add forrestchang/andrej-karpathy-skills` → Claude Code fetches `marketplace.json` 🟢
2. User runs `/plugin install andrej-karpathy-skills@karpathy-skills` → Claude Code reads `plugin.json` 🟢
3. `plugin.json` points to `skills/karpathy-guidelines/` as the skill path 🟢
4. Claude Code loads `SKILL.md` into the LLM's context for the target project 🟢
5. Four behavioral principles become active constraints for all subsequent interactions 🟢

## Alternate Flow — CLAUDE.md Install

1. User copies or appends `SKILL.md` content into the project's `CLAUDE.md` 🟢
2. Claude Code reads `CLAUDE.md` at session start 🟢
3. Same four principles become active — functionally identical to plugin path 🟢

---

## Alternative Flows

- **Skill already in CLAUDE.md, user also installs plugin:** Both load the same content. Duplication is harmless but redundant. 🟡 INFERIDO — no deduplication guard documented.
- **Plugin schema invalid:** Install fails at validation step; no principles activate. Fixed in ADR-006 (`68b67a5`). 🟢
- **Translation variant (`SKILL.zh.md`):** No behavioral difference from English — same rules, different language text. 🟢

## Internal State

This unit is stateless. The skill content is immutable text; the LLM applies it at inference time without persisting any runtime state. 🟢

---

## Observability

No logs, metrics, or traces are emitted. Compliance with the behavioral rules is not mechanically measurable — it depends entirely on LLM inference. 🟡 INFERIDO — see `questions.md`, Q-03.

---

## Risks and Gaps

- 🔴 **No compliance verification** — there is no harness to confirm the LLM actually follows the rules after install (L-03 in `code-analysis.md`)
- 🔴 **Trivial task boundary undefined** — the tradeoff note delegates judgment to the LLM without heuristics (L-01)
- 🟡 **Translation drift** — `SKILL.zh.md` can diverge from `SKILL.md` silently; no CI check exists (L-04)
- 🟡 **Duplicate activation** — if CLAUDE.md and plugin are both active, rules are loaded twice; no deduplication guard
