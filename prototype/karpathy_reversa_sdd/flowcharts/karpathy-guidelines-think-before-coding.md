## Decision Procedure

```mermaid
flowchart TD
    A([Request received]) --> B{Are there hidden assumptions?}

    B -->|Yes — uncertain about scope/format/fields| C["State assumptions explicitly\nIF still uncertain → ask\nDO NOT guess and proceed"]
    C --> D([Wait for user confirmation])
    D --> A

    B -->|Multiple valid interpretations exist| E["Present ALL interpretations\nDO NOT pick silently\nInclude effort estimate per option if helpful"]
    E --> F([Wait for user to choose])
    F --> A

    B -->|Simpler approach exists than what was implied| G["Say so — push back\nPropose the simpler alternative first"]
    G --> H{User wants simpler?}
    H -->|Yes| I([Proceed with simpler approach])
    H -->|No — user confirms original| J([Proceed with original])

    B -->|Something is genuinely unclear| K["STOP\nName exactly what is confusing\nAsk targeted question"]
    K --> L([Wait for clarification])
    L --> A

    B -->|No hidden assumptions — all clear| M([Proceed to implementation])
```

## Anti-Patterns Detected (from EXAMPLES.md)

| Anti-Pattern | Example | Correct Behavior |
|-------------|---------|-----------------|
| Silent scope assumption | "Export all users" — assumes ALL users, ALL fields | Ask: filtered or all? which fields? |
| Silent format assumption | Assumes JSON when user says "export" | Ask: download, background job, or API? |
| Silent optimization choice | "Make search faster" → implements caching + async + indexes | Present 3 interpretations: latency vs. throughput vs. perceived UX |
