## caveman — toggle state machine

```mermaid
stateDiagram-v2
    [*] --> INACTIVE

    INACTIVE --> ACTIVE : trigger words detected\n("caveman mode" / "be brief" / "less tokens" / /caveman)
    ACTIVE --> INACTIVE : "stop caveman" | "normal mode"

    state ACTIVE {
        [*] --> COMPRESSING
        COMPRESSING --> EXCEPTION : security warning\nOR irreversible action\nOR multi-step risk\nOR user confusion
        EXCEPTION --> COMPRESSING : clear section done → resume caveman
    }
```

## caveman — message transformation rules

```mermaid
flowchart TD
    MSG[Incoming response draft] --> EXCEPTION_CHK{Exception trigger?}
    EXCEPTION_CHK -- yes --> FULL_CLARITY[Write full clear language for this section]
    FULL_CLARITY --> RESUME[Resume caveman after section]
    EXCEPTION_CHK -- no --> DROP_ARTICLES[Drop: a / an / the]
    DROP_ARTICLES --> DROP_FILLER[Drop: just / really / basically / actually / simply]
    DROP_FILLER --> DROP_PLEASANTRIES[Drop: sure / certainly / of course / happy to]
    DROP_PLEASANTRIES --> DROP_HEDGING[Drop: hedging phrases]
    DROP_HEDGING --> SHORT_SYN[Use short synonyms: big not extensive, fix not 'implement a solution for']
    SHORT_SYN --> ABBREV[Abbreviate: DB / auth / config / req / res / fn / impl]
    ABBREV --> ARROWS[Causality: X -> Y]
    ARROWS --> KEEP_TECH[Keep: all technical terms exact, code blocks unchanged, error quotes exact]
    KEEP_TECH --> OUTPUT[Output compressed response]
```

## write-a-skill — scaffold loop

```mermaid
flowchart TD
    START([/write-a-skill]) --> GATHER[Gather requirements]
    GATHER --> Q1[What domain/task does it cover?]
    GATHER --> Q2[What specific use cases?]
    GATHER --> Q3[Executable scripts needed?]
    GATHER --> Q4[Reference materials to include?]

    Q1 & Q2 & Q3 & Q4 --> DRAFT[Draft skill]

    subgraph DRAFT_SUB["Draft"]
        D1[Write SKILL.md: frontmatter + instructions]
        D1 --> SIZE_CHK{SKILL.md > 100 lines?}
        SIZE_CHK -- yes --> SPLIT[Split into REFERENCE.md or EXAMPLES.md]
        SIZE_CHK -- no --> D2
        SPLIT --> D2{Deterministic operations?}
        D2 -- yes --> SCRIPTS[Add utility scripts/]
        D2 -- no --> D3[Done]
        SCRIPTS --> D3
    end

    D3 --> REVIEW[Review with user]
    REVIEW --> R1{Covers use cases?}
    R1 -- no --> REVISE[Revise]
    REVISE --> REVIEW
    R1 -- yes --> R2{Missing or unclear?}
    R2 -- yes --> REVISE
    R2 -- no --> DONE([Skill ready])

    subgraph DESCRIPTION_RULES["description field rules"]
        DR1[Max 1024 chars]
        DR2[Third person]
        DR3[Sentence 1: what it does]
        DR4[Sentence 2: Use when specific triggers]
    end
```

## handoff — document generation

```mermaid
flowchart TD
    START([/handoff optionalArg]) --> ARG{Argument provided?}
    ARG -- yes --> FOCUS[Set focus = argument description]
    ARG -- no --> NO_FOCUS[No focus set]

    FOCUS & NO_FOCUS --> COMPACT[Compact entire conversation context]
    COMPACT --> TEMPPATH["Generate temp path: mktemp -t handoff-XXXXXX.md"]
    TEMPPATH --> READ[Read file before writing]
    READ --> WRITE[Write handoff document]

    subgraph WRITE_SUB["Handoff document content"]
        W1[Focus section - if argument provided]
        W2[Key decisions made]
        W3[Current state of work]
        W4[Next steps]
        W5[Skill suggestions for next session]
        W6[References to artifacts - paths/URLs ONLY]
        W_RULE[RULE: never duplicate content already in PRDs/plans/ADRs/issues]
    end

    WRITE --> DONE([Done])
```
