## diagnose — 6-phase state machine

```mermaid
flowchart TD
    START([/diagnose invoked/]) --> P1

    subgraph P1["Phase 1: Build Feedback Loop"]
        FL1[Try: failing unit/integration test]
        FL2[Try: curl/HTTP script vs dev server]
        FL3[Try: CLI invocation + snapshot diff]
        FL4[Try: headless browser / Playwright]
        FL5[Try: replay captured trace]
        FL6[Try: throwaway harness]
        FL7[Try: property/fuzz loop]
        FL8[Try: bisection harness]
        FL9[Try: differential loop]
        FL10[Try: HITL bash script]
        FL1 --> HAVE_LOOP{Loop found?}
        HAVE_LOOP -- no --> FL2 --> FL3 --> FL4 --> FL5 --> FL6 --> FL7 --> FL8 --> FL9 --> FL10
        FL10 --> GIVE_UP{Loop found?}
        GIVE_UP -- no --> STOP_ASK[Ask user for env access / artifacts / prod instrumentation]
        GIVE_UP -- yes --> LOOP_OK
        HAVE_LOOP -- yes --> LOOP_OK[Optimize loop: faster? sharper? deterministic?]
    end

    LOOP_OK --> P2

    subgraph P2["Phase 2: Reproduce"]
        R1{Correct bug reproduced?}
        R2{High enough rate for non-deterministic?}
        R1 -- no --> WRONG_BUG[Rebuild loop for correct bug]
        R1 -- yes --> R2
        R2 -- no --> RAISE_RATE[Loop 100x, parallelise, inject sleeps]
        R2 -- yes --> P3_IN[Capture exact symptom]
    end

    P3_IN --> P3

    subgraph P3["Phase 3: Hypothesise"]
        H1[Generate 3–5 ranked falsifiable hypotheses]
        H2[State prediction for each: IF X THEN Y]
        H3[Show ranked list to user - optional, non-blocking]
        H1 --> H2 --> H3
    end

    H3 --> P4

    subgraph P4["Phase 4: Instrument"]
        I1[Map each probe to a hypothesis]
        I2{Debugger available?}
        I3[Use debugger / REPL]
        I4[Targeted logs at boundary points]
        I5[Tag all logs DEBUG-xxxx]
        I1 --> I2
        I2 -- yes --> I3
        I2 -- no --> I4 --> I5
        PERF{Performance regression?}
        I3 --> PERF
        I5 --> PERF
        PERF -- yes --> MEASURE[Baseline measurement → bisect]
        PERF -- no --> CHANGE_ONE[Change ONE variable at a time per probe]
    end

    CHANGE_ONE --> P5
    MEASURE --> P5

    subgraph P5["Phase 5: Fix + Regression Test"]
        S1{Correct seam exists?}
        S2[Write failing test at seam]
        S3[Watch it fail RED]
        S4[Apply fix]
        S5[Watch it pass GREEN]
        S6[Re-run Phase 1 loop on original scenario]
        S7[Note: no correct seam = architectural gap]
        S1 -- yes --> S2 --> S3 --> S4 --> S5 --> S6
        S1 -- no --> S7 --> S4
    end

    S6 --> P6
    S4 --> P6

    subgraph P6["Phase 6: Cleanup + Post-mortem"]
        C1{Original repro gone?}
        C2[Remove all DEBUG-xxxx logs]
        C3[Delete throwaway prototypes]
        C4[State correct hypothesis in commit/PR]
        C5[Ask: what would have prevented this?]
        C6{Architectural fix needed?}
        C7[Hand off to /improve-codebase-architecture]
        C1 -- no --> RECHECK[Fix more]
        C1 -- yes --> C2 --> C3 --> C4 --> C5 --> C6
        C6 -- yes --> C7
        C6 -- no --> DONE([Done])
        C7 --> DONE
    end
```

## triage — issue state machine

```mermaid
stateDiagram-v2
    [*] --> unlabeled : issue created

    unlabeled --> needs_triage : initial classification

    needs_triage --> needs_info : requires more info from reporter
    needs_triage --> ready_for_agent : fully specified + AFK-ready
    needs_triage --> ready_for_human : needs human judgment
    needs_triage --> wontfix : will not be actioned

    needs_info --> needs_triage : reporter replies

    ready_for_agent --> [*] : agent picks up + resolves
    ready_for_human --> [*] : human implements
    wontfix --> [*] : closed

    note right of ready_for_agent
        Triggers: post Agent Brief comment
    end note

    note right of needs_info
        Triggers: post Triage Notes comment
    end note

    note right of wontfix
        If enhancement: write .out-of-scope/<concept>.md
        If bug: polite explanation + close
    end note
```

## triage — per-issue processing flow

```mermaid
flowchart TD
    START([issue selected]) --> CTX

    CTX[Gather context: full body + comments + labels + .out-of-scope check]
    CTX --> OOS{Matches .out-of-scope entry?}
    OOS -- yes --> SURFACE[Surface prior rejection to maintainer]
    SURFACE --> MAINT_DEC{Maintainer decision}
    MAINT_DEC -- confirm --> CLOSE_OOS[Add to prior_requests + close]
    MAINT_DEC -- reconsider --> DELETE_OOS[Delete/update .out-of-scope entry]
    MAINT_DEC -- disagree --> RECOMMEND
    DELETE_OOS --> RECOMMEND

    OOS -- no --> RECOMMEND
    RECOMMEND[Recommend: category + state + reasoning] --> WAIT{Wait for direction}
    WAIT --> BUG_CAT{Category = bug?}
    BUG_CAT -- yes --> REPRO[Attempt reproduction: trace code + run tests]
    REPRO --> REPRO_RESULT{Result}
    REPRO_RESULT -- success --> GRILL_CHECK
    REPRO_RESULT -- failed --> NEEDS_INFO_SIG[Strong needs-info signal]
    REPRO_RESULT -- insufficient_detail --> NEEDS_INFO_SIG
    NEEDS_INFO_SIG --> GRILL_CHECK

    BUG_CAT -- no --> GRILL_CHECK
    GRILL_CHECK{Needs fleshing out?}
    GRILL_CHECK -- yes --> GRILL[Run /grill-with-docs session]
    GRILL --> APPLY
    GRILL_CHECK -- no --> APPLY

    subgraph APPLY["Apply outcome"]
        RA[ready-for-agent → post Agent Brief] 
        RH[ready-for-human → Agent Brief + non-delegation reason]
        NI[needs-info → post Triage Notes]
        WFB[wontfix + bug → explanation + close]
        WFE[wontfix + enhancement → write .out-of-scope + link + close]
    end

    APPLY --> END([done])
```

## improve-codebase-architecture — deepening workflow

```mermaid
flowchart TD
    START([/improve-codebase-architecture]) --> READ[Read CONTEXT.md + ADRs]
    READ --> EXPLORE[Explore via Explore subagent]
    
    EXPLORE --> SIGNALS{Look for friction signals}
    SIGNALS --> S1[Multi-file conceptual bouncing]
    SIGNALS --> S2[Shallow modules: interface ≈ implementation]
    SIGNALS --> S3[No-locality pure functions]
    SIGNALS --> S4[Seam leaks across modules]
    SIGNALS --> S5[Hard-to-test interfaces]

    S1 & S2 & S3 & S4 & S5 --> DEL_TEST[Apply deletion test to suspects]
    DEL_TEST --> DT_RESULT{Result}
    DT_RESULT -- concentrate → shallow, delete --> SKIP[Skip this module]
    DT_RESULT -- disperse → earning keep --> CANDIDATE[Add to candidates list]

    CANDIDATE --> ADR_CHECK{Contradicts ADR?}
    ADR_CHECK -- yes, friction is real --> MARK[Mark: contradicts ADR-XXXX — worth reopening because...]
    ADR_CHECK -- yes, minor --> OMIT[Omit from candidates]
    ADR_CHECK -- no --> PRESENT

    MARK --> PRESENT
    PRESENT[Present numbered deepening candidates to user]
    PRESENT --> PICK{User picks candidate}

    PICK --> GRILL_LOOP[Grilling loop: walk design tree]
    GRILL_LOOP --> SIDE_EFFECTS{Side effects}
    SIDE_EFFECTS -- new domain term --> UPDATE_CTX[Update CONTEXT.md inline]
    SIDE_EFFECTS -- fuzzy term resolved --> UPDATE_CTX
    SIDE_EFFECTS -- user rejects with load-bearing reason --> OFFER_ADR{Offer ADR?}
    OFFER_ADR -- yes, durable reason --> WRITE_ADR[Write ADR]
    OFFER_ADR -- no, ephemeral reason --> SKIP2[Skip ADR]
    
    GRILL_LOOP --> DEP_CLASS[Classify dependency category]
    DEP_CLASS --> DC1{Category}
    DC1 -- in-process --> MERGE[Merge modules, test directly]
    DC1 -- local-substitutable --> STANDIN[Use local stand-in in tests]
    DC1 -- remote-owned --> PORT[Define port + adapters]
    DC1 -- true-external --> MOCK[Inject port + mock adapter]
    
    MERGE & STANDIN & PORT & MOCK --> DONE([Deepening spec complete])
```

## prototype — branch router

```mermaid
flowchart TD
    START([/prototype]) --> IDENT{Identify question type}
    IDENT -- logic/state model --> LOGIC
    IDENT -- what should it look like --> UI
    IDENT -- ambiguous --> SURROUNDINGS{Surrounding code type}
    SURROUNDINGS -- backend module --> LOGIC
    SURROUNDINGS -- page/component --> UI
    SURROUNDINGS -- unclear + user available --> ASK[Ask user which branch]
    ASK --> LOGIC
    ASK --> UI

    subgraph LOGIC["LOGIC branch"]
        L1[State the question in writing]
        L2[Pick language matching host project]
        L3[Isolate logic in PURE module]
        L3a[reducer: state+action→state]
        L3b[state machine: explicit states+transitions]
        L3c[pure functions over plain data]
        L3d[class with clear method surface]
        L4[Build minimal TUI: clear+re-render on every action]
        L4a[Frame: current state bold + keyboard shortcuts dim]
        L5[One run command via existing task runner]
        L6[Hand over → user drives]
        L7[Capture answer → delete TUI shell OR lift logic to prod]
        L1 --> L2 --> L3
        L3 --> L3a & L3b & l3c & L3d
        L3a & l3c & L3d & L3b --> L4 --> L4a --> L5 --> L6 --> L7
    end

    subgraph UI["UI branch"]
        U0[Default 3 variants, max 5]
        U1{Existing page to host?}
        U1a[Sub-shape A: existing page + ?variant= param]
        U1b[Sub-shape B: /prototype/<name> throwaway route]
        U2[Generate variants: structurally different layout]
        U3[Wire with switcher component]
        U3a[Floating bar: ← label → keyboard ←→ cycles]
        U3b[Hidden in production: NODE_ENV check]
        U4[Hand over → user flips between variants]
        U5[Capture winner + why → delete losers → promote winner]
        U0 --> U1
        U1 -- yes --> U1a
        U1 -- no --> U1b
        U1a & U1b --> U2 --> U3 --> U3a --> U3b --> U4 --> U5
    end
```

## grill-with-docs — interview + inline doc update

```mermaid
flowchart TD
    START([/grill-with-docs]) --> READ[Read CONTEXT.md + ADRs]
    READ --> LOOP{Next question}
    
    LOOP --> CODEANS{Answerable by codebase?}
    CODEANS -- yes --> EXPLORE[Explore codebase instead]
    EXPLORE --> LOOP
    CODEANS -- no --> ASK_ONE[Ask one question at a time]
    ASK_ONE --> USER_REPLY[Await user feedback]

    USER_REPLY --> CONFLICT{Term conflicts with glossary?}
    CONFLICT -- yes --> CALLOUT["Call out: your glossary defines X as Y, but you mean Z — which is it?"]
    CALLOUT --> USER_REPLY

    USER_REPLY --> VAGUE{Term is vague/overloaded?}
    VAGUE -- yes --> SHARPEN[Propose precise canonical term]
    SHARPEN --> USER_REPLY

    USER_REPLY --> SCENARIO{Domain relationship asserted?}
    SCENARIO -- yes --> STRESS[Stress-test with concrete edge-case scenarios]
    STRESS --> USER_REPLY

    USER_REPLY --> CODE_AGREE{User statement matches code?}
    CODE_AGREE -- no --> SURFACE["Surface contradiction: your code does X but you said Y"]
    SURFACE --> USER_REPLY

    USER_REPLY --> RESOLVED{Term/decision resolved?}
    RESOLVED -- yes --> UPDATE_CTX[Update CONTEXT.md immediately — no batching]
    UPDATE_CTX --> ADR_COND{Hard-to-reverse AND Surprising AND Real trade-off?}
    ADR_COND -- ALL THREE --> OFFER_ADR[Offer to write ADR]
    ADR_COND -- missing any --> SKIP_ADR[Skip ADR]

    OFFER_ADR --> LOOP
    SKIP_ADR --> LOOP
    RESOLVED -- no --> LOOP
    
    LOOP -- shared understanding reached --> DONE([Done])
```
