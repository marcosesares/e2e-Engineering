## Overview

This system has no traditional RBAC model — there are no user accounts, roles, or access control lists in code. Instead, there is an **actor model** defined implicitly by skill invariants and triage state machine rules. Four distinct actors are identified.

---

## Actor Definitions

| Actor | Description | How identified |
|-------|-------------|----------------|
| **Maintainer** | Human who triages issues, labels them, and drives the triage state machine | Defined in `triage/SKILL.md` — only actor with write access to issue labels |
| **Reporter** | Human who opens a GitHub/GitLab issue | Defined implicitly in triage flow — triggers `needs-triage` state on issue creation |
| **AFK Agent** | An AI agent that picks up a `ready-for-agent` issue and implements it without human interaction | Defined in `domain.md` and `triage/SKILL.md` — works from Agent Brief, no direct issue tracker access |
| **Human Dev** | A human developer who picks up a `ready-for-human` issue | Implied by `ready-for-human` state — handles HITL work, architectural decisions, external API access |

## Invariants (🟢 CONFIRMED from `triage/SKILL.md`)

1. Only the **Maintainer** may drive state transitions in the triage machine. No automated trigger should change a label without maintainer action, except AI comments posted during a triage skill invocation.
2. All AI-generated comments posted by a skill must begin with `> *This was generated by AI during triage.*` — this identifies the AFK Agent actor in the issue tracker audit trail.
3. The **AFK Agent** reads from the Agent Brief only — it does not access the issue tracker's history, comments, or labels for implementation decisions.
4. **Conflict resolution**: if an issue has conflicting state labels, the skill must flag and ask the Maintainer before acting (🟢 CONFIRMED invariant in triage skill).

## Gaps

🔴 **No audit trail mechanism**: there is no structured log of who applied which label when. GitHub's event history is the only record — not surfaced by any skill.

🔴 **AFK Agent trust boundary undefined**: the system assumes an AFK Agent will not exceed its mandate (implement and close), but there is no technical enforcement. Trust is entirely based on the Agent Brief's behavioral spec.

🟡 **Reporter can close their own issues on GitHub**: this is a GitHub platform permission, not surfaced or constrained by any skill.
